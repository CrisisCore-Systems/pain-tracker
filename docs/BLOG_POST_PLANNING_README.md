These statistics are sourced directly from the Pain Tracker codebase and should be cited across blog posts:

### Codebase Metrics
- **31,700+ lines of code** (production TypeScript)
- **580+ TypeScript files** in `/src`
- **96 documentation files** (comprehensive guides)
- **90%+ test coverage** (Vitest + Playwright)
- **420KB gzipped bundle** (optimized from 2.6MB)

### Feature Completeness
- **44+ anatomical locations** for pain tracking (26 general + 18 fibro-specific)
- **19+ symptom quality types** (sharp, burning, tingling, etc.)
- **7-step assessment workflow** (trauma-informed design)
- **25+ empathy metrics** (heuristic-based analysis)
- **79+ keyword patterns** (emotion detection)

### Technology Stack
- **React 18** + **TypeScript** + **Vite**
- **Zustand** for state management
- **IndexedDB** + **localStorage** (dual-persistence)
- **Service Workers** (offline-first PWA)
- **Vitest** + **Playwright** + **Stryker** (testing)

### Compliance & Security
- **HIPAA-aligned** practices
- **WorkSafe BC** compliant exports
- **WCAG 2.1 AA** accessibility
- **ACR 2016** fibromyalgia criteria
- **AES-256** encryption at rest
