# Protective Boundary Statement

This document defines where the protective sanctuary begins and ends.

## What We Intend To Protect

- Class A health records stored locally.
- User autonomy over exports and disclosures.
- Predictable access to safety tools during crisis states.

## What We Explicitly Do Not Guarantee

- Protection from a compromised operating system or malware.
- Protection from a malicious extension with full page access.
- Protection from device seizure when the device/session is already unlocked.

## System Boundary

- In-scope: local app behavior, local storage controls, local-first exports.
- Out-of-scope: third-party systems not controlled by this repository.

## Claims Discipline

- Do not claim protections that are not test-backed.
- If a claim cannot be verified with evidence, remove or downgrade the claim.

## Review Cadence

- Re-approve this statement each release touching security-critical or crisis-critical paths.
