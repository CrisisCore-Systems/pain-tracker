# Screenshot Portfolio

This page is the landing document for Pain Tracker screenshots used in marketing, docs, and launch materials.

## What lives here

- One canonical place to find screenshot guidance
- Links to the capture scripts and route references
- A summary of the current portfolio phases without turning the README into a marketing catalog

## Capture Commands

```powershell
npm run screenshots:portfolio
npm run screenshots:portfolio:phase1
npm run screenshots:portfolio:phase2
npm run screenshots:portfolio:phase3
```

To test demo routes locally:

```powershell
npm run dev
```

Then open the local demo route documented in the marketing guides.

## Portfolio Summary

Phase 1 covers essential product proof such as pain entry, export, and privacy settings.

Phase 2 covers growth-oriented visuals such as analytics, body-map interaction, offline behavior, and responsive layouts.

Phase 3 covers advanced visuals such as trauma-informed mode, process diagrams, architecture views, and crisis support states.
