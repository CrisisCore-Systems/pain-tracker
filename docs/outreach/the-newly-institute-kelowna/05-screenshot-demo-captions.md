# Three Screenshot Demo Captions

## Screenshot 1: Dashboard

### What The Viewer Is Seeing

The dashboard shows the user's recent pain tracking activity in one place. It may include recent entries, pain severity trends, common pain locations, symptom patterns, and quick access to logging or review areas.

Exact screenshot elements should be confirmed against the current app screenshot before sending. If the screenshot shows demo or sample data, label it clearly as demo data.

### Why It Matters To A Clinician Or Program Coordinator

The dashboard can help a patient move from scattered memory to a more consistent summary of what has been happening between appointments. For program staff, it shows that the tool is meant to support patient recall and appointment preparation rather than replace clinical documentation.

### Suggested Caption Text

Dashboard view: A private patient-controlled overview of recent pain tracking activity, designed to help users review symptom patterns before an appointment.

## Screenshot 2: New Pain Entry Screen

### What The Viewer Is Seeing

The new pain entry screen lets the user record a daily or event-based pain entry. The screen may include fields for pain severity, pain location, symptoms, triggers, medication notes, functional impact, sleep context, and free-text notes.

Keep the screenshot focused on general fields and avoid showing sensitive personal details. Use fictional demo data only.

### Why It Matters To A Clinician Or Program Coordinator

This view shows the low-friction daily capture point. It helps explain how a patient can record details close to when symptoms occur, rather than trying to reconstruct everything later in a clinical appointment.

### Suggested Caption Text

New entry view: Patients can record pain severity, location, symptoms, triggers, medication notes, and functional impact while details are still fresh.

## Screenshot 3: Export Or Report Screen

### What The Viewer Is Seeing

The export or report screen shows how a user can create a patient-controlled summary or file for review. It may include date range selection, report format choices, and download options.

The screenshot should make clear that export is user initiated. Do not imply automatic clinic sharing, insurer sharing, employer sharing, or WorkSafeBC submission.

### Why It Matters To A Clinician Or Program Coordinator

This view shows the boundary between private tracking and optional sharing. It helps reassure staff that PainTracker.ca does not require clinic integration for a small pilot and that the patient controls what leaves the app.

### Suggested Caption Text

Export view: Patients choose the date range and report format, then decide whether to save, print, email, or bring the summary to an appointment.
